# セットアップページの修正と機能追加のウォークスルー

セットアップページ（`/`）において、URL生成の不具合修正と、表示位置指定機能の完全な修正を行いました。

## 変更内容

### UI Components

#### [MODIFY] [page.tsx](../../src/app/page.tsx)
- **バグ修正 (依存配列の追加)**:
  - 表示位置（`pos`）を選択しても生成URLが更新されていなかった問題を修正しました。
  - `useEffect` の依存配列に `pos` を追加したことで、ボタン選択と同時に `overlayUrl` が再計算されるようになりました。
- **URL生成ロジックの修正**:
  - `window.location.pathname` をベースにしたパス計算により、サブパス環境（GitHub Pages等）での正確なURL生成を保証しました。
- **表示位置指定UIの追加**:
  - ラジオボタン形式のUIを追加し、「左側に表示」「右側に表示」を直感的に選択できるようにしました。

## 検証結果

### 動作確認
- **リアルタイム反映**: 「右側に表示」をクリックすると、表示されているオーバーレイURLに即座に `&pos=right` が付与されることを確認しました。
- **コピー動作**: コピーしたURLにパラメータが含まれていること、およびそのURLをOBS等で使用した際に正しく右側にコメントが表示されることを確認しました。
