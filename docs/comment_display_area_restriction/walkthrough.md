# 修正内容の確認 - コメント表示エリアの制限とパラメータ化

画面全体の上部および下部の余白を制限し、その値をURLパラメータで動的に指定できるようにしました。

## 変更内容
表示領域を制限する `absolute` 配置のコンテナを導入し、`overflow: hidden` によって境界線でのコメントの切り取り（クリッピング）を確実にしました。また、余白の値をURLパラメータで指定できる機能を追加しました。

### 追加されたURLパラメータ
- `top`: 上部の余白（vh単位、0〜100）。デフォルト：`10`
- `bottom`: 下部の余白（vh単位、0〜100）。デフォルト：`10`

例: `?top=20&bottom=5` と指定すると、上部に20%、下部に5%の表示不可領域が作成されます。

### 対象ファイル
- [overlay/page.tsx](../../src/app/overlay/page.tsx)
- [sample/page.tsx](../../src/app/sample/page.tsx)

## 検証結果
ブラウザ（`/sample` ページ）にて以下の動作を確認しました。
- パラメータ `top`, `bottom` の値に応じて、リアルタイムに表示領域のサイズと位置が変化すること。
- 指定された境界線でコメントが正しく切り取られ、余白領域にはみ出さないこと。
- パラメータ未指定時は、従来のデフォルト値（上下10%）が適用されること。
- サンプルのヘッダーやエラー表示も、`top` の値に合わせて動的に位置が調整されること。
