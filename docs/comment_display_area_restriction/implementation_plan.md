# 実装計画 - コメント表示エリア制限のパラメータ化

画面全体の上部および下部の余白を、URLパラメータで自由に変更できるようにします。

## 概要
URLパラメータ `top` および `bottom` を読み取り、コメント表示制限用コンテナの高さ（`top`, `bottom`）に反映させます。パラメータが指定されない、または不正な値の場合は、デフォルト値として `10`（10vh）を使用します。

## 変更内容

### [共通ロジック]
- `useSearchParams` から `top` と `bottom` を取得します。
- `parseInt` 等で数値化し、適切な範囲（例: 0〜100）に制限した上で変数に格納します。

### [Overlay & Sample Pages]
#### [MODIFY] [page.tsx](../../src/app/overlay/page.tsx)
#### [MODIFY] [page.tsx](../../src/app/sample/page.tsx)
- パラメータ `top`, `bottom` を取得・計算するロジックを追加。
- 制限用コンテナ（`absolute` コンテナ）の `style` 属性に、計算された値を設定します。
  - `style={{ top: \`${top}vh\`, bottom: \`${bottom}vh\` }}`
- ヘッダーやエラー表示の位置（`top`）も、`top` パラメータに応じて動的に調整します（例: `top: \`${top + 1}vh\` `）。

## 検証プラン
### ブラウザでの確認
- `http://localhost:3000/sample?top=20&bottom=5` にアクセスし、上部が20%、下部が5%の余白になっていることを確認。
- パラメータがない場合にデフォルトの10%が適用されることを確認。
- 負の値や大きすぎる値（100以上）などの異常値が入力された際に、デフォルト値または安全な範囲にフォールバックされることを確認。
